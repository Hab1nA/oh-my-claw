# 适合展示大语言模型效果的实验任务

基础工具任务只能证明 Mini-OpenClaw 的执行流程可以运行，不能充分体现大语言模型的价值。答辩或论文实验中，建议重点展示以下任务。

## 任务一：需求提取与风险分析

```text
请读取 workspace/project_brief.txt，提取系统的核心功能、非功能需求、潜在风险，并按照 Markdown 表格输出
```

展示点：

- `read_file` 工具只负责读取原始材料。
- 大模型负责理解项目说明，并将非结构化文本整理为结构化表格。
- 该任务可用于说明 LLM 的语义理解和信息抽取能力。

## 任务二：用户反馈分类与优先级排序

```text
请读取 workspace/customer_feedback.txt，把用户反馈分为正向反馈、负向反馈和改进建议，并给出优先级排序
```

展示点：

- 反馈文本包含褒贬混合内容。
- 大模型需要判断情绪、识别问题类型、推断优先级。
- 简单关键词规则很难稳定完成这个任务。

## 任务三：论文段落生成

```text
请读取 workspace/research_notes.txt，生成一段适合论文“实验结果与分析”章节使用的文字，要求包含实验现象、原因分析和系统不足
```

展示点：

- 大模型将零散实验记录组织成正式论文语言。
- 输出可直接用于课程论文草稿。
- 该任务体现文本生成和表达组织能力。

## 任务四：汇报稿生成并保存

```text
请读取 workspace/project_brief.txt，生成一份面向老师汇报的 5 分钟展示稿，并写入 workspace/presentation_script.md
```

展示点：

- 该任务需要读取材料、生成文本、再调用 `write_file` 保存结果。
- 可以展示多步 Agent Loop。
- 日志中会出现至少一次读取工具调用和一次写入工具调用。
